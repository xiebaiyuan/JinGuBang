# 工具贡献规范

本仓库以“按场景分类的可复用脚本”为核心，新增工具请遵循以下约定。

## 1. 命名规范

- 脚本使用 `snake_case`。
- 推荐模式：`verb_object.py` / `verb_object.sh`。
- 避免无语义缩写与版本后缀（如 `tool_v7_final.py`）。

## 2. CLI 参数规范

- 必须支持 `-h/--help`。
- 输入参数优先 `-i/--input`。
- 输出参数优先 `-o/--output`。
- 破坏性操作需提供 `--dry-run` 或 `--apply` 显式确认。

## 3. 错误处理

- 参数错误、路径不存在、依赖缺失要明确报错。
- 错误返回码非 0。
- 报错文本应包含可执行建议。

## 4. 依赖策略

- 优先 Python 标准库。
- 若使用第三方依赖，需在 README 标明并提供安装命令。
- 可选依赖需有降级路径或清晰提示（例如 Pillow/PyYAML）。

## 5. 文档要求

- 每个目录 README 使用统一表格：

`工具名 | 用途 | 使用示例 | 依赖 | 平台支持 | 风险级别`

- 新工具必须在 `TOOLS_INDEX.md` 添加关键词入口。

## 6. 安全与边界

- 网络工具仅用于授权环境，README 必须注明。
- 删除/移动类工具默认安全模式，不应默认执行不可逆操作。

## 7. 最小验证

- 新增 Python 脚本至少通过：`python3 script.py -h`
- 每个新工具至少验证：
  - 1 条成功路径
  - 1 条失败路径（无效输入）
